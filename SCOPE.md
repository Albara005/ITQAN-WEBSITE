# نطاق المشروع — منصة «إتقان» للخدمات الأكاديمية

> وثيقة تنظيمية تجمع **فكرة المشروع** + **ما تم إنجازه فعليًا** + **خارطة الطريق** للوصول للرؤية الكاملة.
> آخر تحديث: 29 يونيو 2026

---

## 1) الرؤية

منصة إلكترونية للطلاب الجامعيين (قابلة للتوسّع لكل الطلاب) تقدّم خدمات أكاديمية بأسعار رمزية، مع التركيز على **الجودة، السرعة، وسهولة الاستخدام**.

**الهدف:** توفير وقت الطالب وتسهيل المذاكرة عبر تحويل الملفات الطويلة والملاحظات إلى محتوى مختصر واضح سهل الفهم — وإظهار الاحترافية *قبل* طلب الخدمة.

|---|--------|------------------|
| 1 | إعداد ملخصات دراسية احترافية | أساسية |
| 2 | إعداد عروض PowerPoint احترافية | أساسية |

## 3) تدقيق الحالة الحالية (ما تم بناؤه فعليًا)

الموقع الحالي = **صفحة هبوط تسويقية (Landing Page)** من ملف واحد `index.html` (HTML/CSS/JS، بدون أطر عمل)، تُقدّم عبر خادم Node محلي `serve.cjs`.

> ✅ = مُنجز ومُتحقق منه · ⚠️ = موجود لكن يحتاج مواءمة مع الرؤية · ❌ = غير موجود بعد

| قسم «كيف يعمل» (3 خطوات) | ✅ | — |

### ⚠️ يحتاج مواءمة مع الرؤية
| المكوّن | الوضع الحالي | المطلوب |
|--------|--------------|----------|
| الخدمات المعروضة | الملخصات فقط ضمنيًا | إبراز **كل الخدمات** (ملخصات/PowerPoint/أسئلة) |
| الأسعار | تعرض «تواصل معنا» | أسعار صريحة **بالريال العماني** (انظر §6) |
| قسم «المواد» | تخصصات دراسية | يمكن تحويله/إضافة قسم **«خدماتنا»** |

### ❌ غير موجود (مطلوب للرؤية الكاملة)
- **رفع الملفات** (PDF / Word / صور / محاضرات).
- **اختيار الخدمة** والتدفق متعدد الخطوات (خدمة → ملفات → إضافات → إرسال).
- **الإضافات الاختيارية** (أسئلة مراجعة ☐، تصميم احترافي ☐، تسليم سريع ☐).
- **باك-اند**: استقبال الطلبات وتخزينها (قاعدة بيانات).
- **لوحة تحكم الإدارة**: عرض الطلبات + معلومات العميل + إدارة الملفات.
- **الإشعارات الفعلية** (بريد / واتساب) عند وصول طلب.
- **الدفع** (إن لزم) أو تأكيد يدوي.

---

## 4) الفجوة باختصار

> الحالي **يبيع الفكرة بشكل احترافي** (واجهة تسويقية ممتازة)، لكنه **لا ينفّذ العملية** بعد: لا يرفع ملفات، ولا يخزّن طلبًا، ولا توجد لوحة إدارة. نموذج الطلب حاليًا **واجهة فقط** (رسالة نجاح وهمية بدون خادم).

نحتاج الانتقال من **«صفحة هبوط»** إلى **«منصة تشغيلية»**.

## 5) خارطة الطريق (المراحل)

### المرحلة 0 — الأساس التسويقي ✅ (مُنجزة)
صفحة الهبوط الحالية بكل أقسامها والهيرو السينمائي والشعار.

### المرحلة 1 — أقل منتج قابل للتشغيل (MVP) 🚧 *قيد التنفيذ*
الهدف: استقبال طلب حقيقي مع ملفات، وتخزينه، وتنبيه الإدارة.
- [x] الأسعار بالريال العماني (ملخصات 1/2/3 + إشارة لعروض PowerPoint والإضافات).
- [x] تطوير نموذج الطلب إلى **تدفق 4 خطوات**: الخدمة → الملفات → الإضافات → بياناتك (مع ملخص للطلب).
- [x] **رفع الملفات** (PDF/Word/PowerPoint/صور، حتى 25MB، سحب وإفلات) — تخزين محلي على القرص.
- [x] **باك-اند** يستقبل الطلب ويحفظه (`serve.cjs` + busboy + `orders/orders.json` + مجلد لكل طلب).
- [x] صفحة/حالة شكر مع **رقم طلب** (مثال: `ITQ-260629-XXXX`).
- [x] تم التحقق end-to-end (إرسال طلب فعلي + تخزينه مع ملفاته).
- [x] **إشعار بريد** للإدارة عند كل طلب (nodemailer) — **اختياري**: يعمل فور ضبط بيانات SMTP (env)، وإلا يُسجَّل في الطرفية.
- [ ] إشعار **واتساب** تلقائي (يحتاج واجهة برمجية مدفوعة) — حاليًا يوجد زر «واتساب» في لوحة الإدارة للتواصل اليدوي.
- [ ] نقل التخزين من المحلي إلى **سحابي** عند النشر.

> ملاحظة: الباك-اند الحالي **محلي** (يعمل على جهازك). كافٍ للتجربة والتشغيل التجريبي؛ يُنقل لخدمة سحابية (Supabase/Firebase) قبل النشر العام.

### المرحلة 2 — لوحة تحكم الإدارة ✅ (مُنجزة)
الرابط: `http://localhost:8000/admin` — كلمة المرور الافتراضية: `itqan2026` (تُغيّر عبر env `ADMIN_KEY`).
- [x] **تسجيل دخول** محمي بكلمة مرور (كل نقاط الإدارة تتطلّب المفتاح).
- [x] **إحصائيات** + جدول الطلبات (الرقم، التاريخ، الخدمة، العميل، الملفات، الحالة).
- [x] لوحة تفاصيل لكل طلب + **معاينة/تحميل** الملفات المرفوعة (عبر مصادقة).
- [x] **تحديث حالة الطلب** (جديد → قيد العمل → جاهز → مُسلّم) ويُحفظ فورًا.
- [x] أزرار **تواصل سريع** مع العميل (واتساب / بريد).
- [x] تم التحقق end-to-end (دخول، عرض، تغيير حالة، تحميل ملف، حماية 401).

### المرحلة 3 — التوسّع والتجربة
- [ ] حساب للطالب + سجل طلباته.
- [ ] خرائط ذهنية / Infographics كخدمات.
- [ ] دفع إلكتروني.
- [ ] تقييمات حقيقية من العملاء + معرض أعمال موسّع.

## 7) المتطلبات التقنية المقترحة

| الطبقة | الوضع الحالي | المقترح للمراحل القادمة |
|--------|---------------|--------------------------|
| الواجهة (Frontend) | HTML/CSS/JS بملف واحد | كافٍ للـ MVP؛ يمكن لاحقًا React/Vite عند التوسّع |
| الخادم (Backend) | لا يوجد (خادم ملفات ثابت فقط) | API لاستقبال الطلبات (Node/Express أو خدمة جاهزة) |
| قاعدة البيانات | لا يوجد | لتخزين الطلبات والعملاء |
| تخزين الملفات | لا يوجد | تخزين سحابي للملفات المرفوعة |
| الإشعارات | رسالة نجاح وهمية | بريد / واتساب فعلي |
| الاستضافة | محلي (`localhost:8000`) | استضافة عامة + نطاق |

## 8) تدفق المستخدم المستهدف

1. **الدخول** → يشاهد الخدمات، أمثلة الأعمال، طريقة العمل، الآراء.
2. **اختيار الخدمة** (ملخص / PowerPoint / مركّب).
3. **رفع الملفات** + اسم المادة + الموعد النهائي + ملاحظات.
4. **اختيار الإضافات** (أسئلة مراجعة / تصميم احترافي / تسليم سريع).
5. **إرسال الطلب** → يصل للوحة الإدارة + يستلم العميل تأكيدًا.

---

## 9) الخطوات التالية الموصى بها

1. **اعتماد هذه الوثيقة** كمرجع للنطاق.
2. البدء بـ **المرحلة 1 (MVP)**، وأول أولوية: **رفع الملفات + باك-اند لتخزين الطلب**.
3. مواءمة محتوى صفحة الهبوط (الخدمات + الأسعار بالريال العماني).
4. تحديد أداة الباك-اند/التخزين المفضّلة قبل البدء بالتنفيذ.
